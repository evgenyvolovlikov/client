# NgAlg

Современное Angular-приложение, построенное с упором на строгое качество кода, автоматизацию и архитектурную методологию Feature-Driven Design (FSD).

## 🛠 Стек технологий

- **Фреймворк:** Angular 19+ (с использованием Standalone-компонентов и Signals)
- **Пакетный менеджер:** pnpm
- **Контроль качества и линтинг:**
    - ESLint (с плагинами `@angular-eslint` и `eslint-plugin-boundaries`)
    - Prettier (автоматическое форматирование кода)
    - Stylelint (проверка и сортировка CSS/SCSS стилей)
- **Автоматизация Git:** Husky и lint-staged
- **CI/CD пайплайны:** GitHub Actions

## 🏗 Архитектура (FSD)

Проект спроектирован по методологии **Feature-Driven Design (FSD)**.
Направление импортов строго контролируется линтером (правило `boundaries/dependencies`), запрещая слоям снизу импортировать логику из слоев выше.

Структура слоев (от высшего к низшему):

- `app/` — Настройки приложения, глобальные стили, роутинг и провайдеры.
- `pages/` — Страницы приложения (композиция из виджетов и фич).
- `widgets/` — Самостоятельные крупные блоки интерфейса (объединяют фичи и сущности).
- `features/` — Действия пользователя и бизнес-логика, приносящая ценность.
- `entities/` — Бизнес-сущности (например: Пользователь, Статья) с их состоянием.
- `shared/` — Переиспользуемые компоненты, UI-кит, пайпы, директивы и утилиты.

## 🚀 Быстрый старт

### Требования

Убедитесь, что у вас установлены **Node.js версии 22+** и **pnpm**.

```bash
# Проверить версии инструментов перед запуском
node -v # v22.x.x
pnpm -v # v10.x.x
```

### Установка

Клонируйте репозиторий и установите все зависимости:

```bash
git clone https://github.com
cd client
pnpm install
```

### Запуск локального сервера

Запустите проект в режиме разработки:

```bash
pnpm start:dev
```

Приложение будет доступно по адресу `http://localhost:4200/`. Страница автоматически обновляется при сохранении изменений в коде.

## 🎛 Доступные скрипты

- `pnpm start:dev` — Запуск локального сервера разработки.
- `pnpm build:prod` — Сборка оптимизированного продакшн-билда.
- `pnpm validate` — **Комплексная проверка кода**. Запускает проверку ESLint, Stylelint и Prettier. Обязательно выполняйте перед отправкой кода в репозиторий!
- `pnpm format:fix` — Автоматически форматирует все файлы проекта через Prettier.
- `pnpm lint:fix` — Автоматически исправляет доступные ошибки линтера в TypeScript и HTML.
- `pnpm lint:styles:fix` — Автоматически исправляет и сортирует свойства в файлах стилей.

## ⚓ Правила оформления коммитов

В репозитории настроен `commitlint` и Husky для проверки сообщений коммитов по стандарту **Conventional Commits**. Сообщение должно строго соответствовать шаблону:

```text
тип: описание изменений
```

**Разрешенные типы коммитов:**
`feat` (фича), `fix` (багфикс), `docs` (документация), `style` (стили), `refactor` (рефакторинг), `perf` (производительность), `test` (тесты), `build` (сборка/пакеты), `ci` (настройки CI/CD), `chore` (рутина/husky/линтеры), `revert` (откат изменений).

_Пример правильного коммита:_ `feat: add custom font assets`
